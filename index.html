fig

Your classroom should be a reflection of your personal learning style. So we made classrooms customizable.

Fig rooms provide students with a modularized real-time collaborative learning environment. Fig allows students to build a virtual ‘classroom’ where they can use built-in learning modules to work and study together on a variety of subjects.

Because real-time collaboration is integral to our platform, we leveraged Google Firestore’s listeners and callbacks to instantly sync application data across users. Data was modeled shallowly for read efficiency and scalability. Our stack included Express and the Twilio API for video chat capabilities. On the front-end, we used React for its modularity and reusability. In developing the canvas module, we ran into the challenge of optimizing rendering and decided to batch the HTML Canvas micro-strokes (pixel to pixel) as larger macro-strokes objects to elevate the strain on database reads, which improved performance. Another consideration during development was protecting code editors against malicious or non-terminating code. We set up execution time-outs and sanitized code before evaluation. Material UI and React Drag and Drop libraries were used to help provide a clean aesthetic and great fig user experience.

  • Technology Stack:
  • Google Firebase Firestore
  • React
  • Express
  • Semantic UI / Material UI

EYEZ

A staring contest where you can play anyone in the world.

EYEZ was built using React Native in conjuction with the Expo XDE. Expo gave access to native device features (such as the iPhone X true depth sensor) with ease. The True Depth sensor provides 50 blendshape coeffecients which map the expression of the human face. I leveraged these coeffecients to tell whether a person was blinking or smiling. Socket.io was used to enable users to complete live in realtime.

  • Technologies Used:
  • React Native
  • Socket.io
  • iPhone X True Depth Sensor
